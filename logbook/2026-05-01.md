## License detection

`LICENSE` was the human-readable CC-BY-4.0 deed (~1.5 KB), not the
canonical legal code. `licensee` (GitHub SPDX detector) fuzzy-matches
canonical templates; the deed never matched. Same problem in
`LICENSE-CODE` — custom "Scope" preamble plus multi-line copyright
broke MIT match. Replaced both with canonical SPDX-conformant texts.
Author institution and ORCID metadata moved to a new `LICENSING.md`.
GitHub API `/license` now returns `key: cc-by-4.0`.

## Line endings

`core.autocrlf=true` was active globally despite the workstation being
Linux-only. Fixed with per-repo `core.autocrlf=input` plus a
`.gitattributes` file authoritative for all source/text formats and
binary assets.

## Reproducibility flag

Two active instruction files (`qwen2.5-7b-fp16/README.md` and
`run_plan_a.sh`) hardcoded `AMD_SERIALIZE_KERNEL=3`. Current PyTorch
silently rejects 3, defeating the intended kernel serialization.
Updated to `=1` with inline comment. Historical artefacts
(pre-discovery logbook entries, paper draft documenting the discovery,
hypothesis-test scripts) intentionally left unchanged — audit trail.
