# UPS acceptance test (ARMAC Office On-Line PF1 2000VA)

### Goal
Validate that the AI workstation operates correctly on the UPS and that the UPS provides voltage regulation in INV mode.

### Procedure
1. Connected UPS directly to a grounded wall outlet.
2. Connected workstation PSU (IEC C20) to UPS output using IEC C13→C20 16A cable.
3. Switched UPS to INV (double conversion) mode.
4. Observed input/output voltage and mode LEDs on UPS display.

### Results
- Workstation runs normally when powered via UPS.
- UPS INV mode provides regulated output (~230V) while input is ~239–240V.
- In BYPASS mode, output tracks input (~239–240V).
- Noted faint high-frequency inverter noise in INV mode; absent in BYPASS (expected behaviour for online UPS).

### Status
PASS ✅

## ROCm installation causing black screen on Kubuntu (multi-GPU workstation)

### System configuration on ROCm install

Hardware:

- CPU: AMD Ryzen 9 9950X3D
- Motherboard: ASUS ProArt X870E-Creator WiFi
- GPU: 2 x Radeon AI PRO R9700
- PSU: FSP PTM PRO 1650W
- UPS: ARMAC Office On-Line PF1 2000VA
- Monitor connected to the first discrete GPU (HDMI)

Software:

- OS: Kubuntu 24.04 LTS
- Kernel: 6.17.0-14-generic
- Desktop: KDE Plasma (X11 session)


---

# Problem observed

After installing ROCm and rebooting:

- System passed BIOS POST
- Linux kernel started loading
- The following message appeared on screen:

RDSEED32 is broken. Disabling the corresponding CPUID bit.

Immediately after this message:

- The screen went completely black
- No login screen appeared
- No keyboard shortcuts produced output
- The GPU appeared to stop initialising the display
- The system appeared to boot normally
- This occurred even when:
  1) Monitor connected to the first discrete GPU (HDMI);
  2) Monitor connected to the second discrete GPU (HDMI);
  3) Monitor connected to iGPU;
  4) Monitor connected to the first discrete GPU, only **one GPU** was installed.
  5) Monitor connected to iGPU, **no GPU** was installed.

---

# Additional symptoms

The system appeared to continue booting, but without display output.

Attempts to switch to a terminal using:

Ctrl + Alt + F3


did not produce visible output.

---

# GRUB access problem

Initially GRUB menu was not visible during boot.

The motherboard BIOS was configured with:

Fast Boot = Enabled

# BIOS investigation

Entering the BIOS after a safe POST revealed that the system was booting in **safe mode**.

Key BIOS observations:

- BIOS version: 1715
- PCIe configuration detected correctly
- GPU hardware was functioning normally

---

# BIOS configuration changes

To enable GRUB to appear and debug the Linux boot process, the following BIOS settings were modified.

### Boot settings

Fast Boot → Disabled

Test command:

This change allowed GRUB to appear reliably during startup.

### PCIe configuration (for multi-GPU stability)

Above 4G Decoding → Enabled
Re-Size BAR Support → Enabled

### PCIe power management

CPU PCIe ASPM Mode Control → Disabled

This avoids link power management issues with multiple GPUs.

### Link mode (optional but recommended)

PCIEX16_1 Link Mode → Gen4
PCIEX16_2 Link Mode → Gen4

---

# Recovery procedure

Once GRUB became accessible (Shift key):

1. Boot entry was edited.
2. Kernel parameter `nomodeset` was added:

linux ... nomodeset

3. System booted successfully into KDE.

This confirmed the issue was related to the GPU driver stack.

---

# Removal of problematic ROCm installation

After gaining system access:

sudo amdgpu-install --uninstall
sudo apt autoremove

System rebooted successfully, and the graphical environment started normally.

---

# Root cause (probable)

The issue appears to be caused by installing ROCm with:

--usecase=graphics,rocm

which installs a full graphics stack that conflicts with the existing KDE graphics configuration on Kubuntu 24.04.

This resulted in the amdgpu driver taking control of display initialisation in a way incompatible with the current desktop configuration.

---

# Recommended installation approach

For AI/compute workloads, it is preferable to install ROCm **without the graphics stack**:

sudo amdgpu-install -y --usecase=rocm --no-dkms


This installs:

- ROCm runtime
- HIP compute libraries
- GPU compute support

without replacing the desktop graphics driver.

# Lessons learned

For Kubuntu + ROCm + multi-GPU systems:

1. Disable **Fast Boot** in BIOS to ensure GRUB visibility.
2. Install ROCm compute stack **without graphics drivers**.
3. Test ROCm with **one GPU first** before installing additional GPUs.
4. Always verify the kernel driver (`amdgpu`) works before installing ROCm.
